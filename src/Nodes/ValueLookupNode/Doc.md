# Value Lookup 节点

## 1. 节点说明

Value Lookup 根据输入值在**规则表**中查找第一条匹配条件，输出该行配置的 Value。适合「输入码 → 输出参数」映射，如状态码转文案、通道号转 DMX 值、枚举到预设字符串等。

无匹配时输出**空字符串**。每次匹配成功会附带 `_matchedRow`（行号）与 `_pulse`（递增）。

## 2. 端口说明

### 输入

| 端口 | 名称 | 数据类型 | 说明 |
|------|------|----------|------|
| 0 | INPUT 0 | VariableData | 用于与 Condition 列比较的值 |

### 输出

| 端口 | 名称 | 数据类型 | 说明 |
|------|------|----------|------|
| 0 | OUTPUT 0 | VariableData | 命中行的 Value（整数/浮点/布尔/字符串自动解析） |

## 3. 界面说明

规则表：

| 列 | 说明 |
|----|------|
| Condition | 匹配条件（见 Distribute 节点相同语法：`*`、`==`、`>`、`<` 等） |
| Value | 命中后输出的值 |
| Action | 删除该行 |

- **Add Rule**：新增一行。  
- 按**从上到下**顺序，**第一条**匹配的规则生效。

## 4. 使用说明

1. 自上而下填写 Condition 与 Value（更具体的规则放上面）。
2. 将待查值接到 INPUT。
3. 从 OUTPUT 取映射结果。

规则保存在 `rules`：`[{ "condition": "...", "value": "..." }, ...]`。

## 5. 示例

| Condition | Value | 输入 3 时输出 |
|-----------|-------|----------------|
| `== 1` | `sceneA` | — |
| `== 2` | `sceneB` | — |
| `== 3` | `sceneC` | `sceneC` |
| `*` | `default` | 兜底 |

控台编号接入 INPUT，输出字符串接场景加载或 OSC 地址生成。
