# Serial Port 节点

## 1. 节点说明

串口收发节点，用于与外部设备通过串口通信。可自动打开串口、发送数据，并将收到的数据从输出端口送出。适合控制舞台设备、传感器、单片机等串口设备。

## 2. 端口说明

### 输入

| 端口 | 名称 | 说明 |
|------|------|------|
| 0 | VALUE | 收到数据后写入发送内容，并立即发送 |
| 1 | TRIGGER | 收到数据后立即按当前界面内容发送（不修改发送文本） |

### 输出

| 端口 | 名称 | 说明 |
|------|------|------|
| 0 | REC | 收到串口数据时输出 VariableData（含多种编码格式的字段） |

## 3. 界面说明

- **Port**：选择串口；可点 **Refresh Ports** 刷新本机可用串口列表。
- **Baud Rate**：波特率（1200～921600，默认 9600）。
- **Data Bits / Parity / Stop Bits**：数据位、校验、停止位；修改后会重新打开串口。
- **Value**：待发送的文本内容。
- **Format**：发送编码，可选 HEX、UTF-8、ASCII。
- **Connected / Disconnect**：连接状态指示（只读，绿色为已连接）。
- **Send**：手动发送；仅连接成功时可点。

## 4. 使用说明

1. 将节点拖入画布，选择正确串口与波特率；节点加载后会尝试自动打开串口。
2. 在 **Value** 填写要发送的内容，选择 **Format**，点击 **Send** 或从 **VALUE** / **TRIGGER** 端口触发发送。
3. 将 **REC** 连到下游节点，处理接收数据。
4. 修改串口参数后节点会重新打开串口；工程保存时会记住端口、波特率、帧格式等设置。

**外部控制路径**（完整地址格式：`/dataflow/{父级别名}/{节点ID}{相对路径}`）：

| 相对路径 | 作用 |
|----------|------|
| `/port` | 设置串口名（如 `COM3`） |
| `/baudRate` | 设置波特率（整数） |
| `/value` | 设置发送内容 |
| `/connected` | 连接状态（只读反馈） |
| `/send` | 触发发送（命令） |

以上路径同时支持 OSC / 全局事件总线命令；`/send` 执行时会短暂反馈 `true` 再恢复 `false`。

## 5. 示例

- **按钮触发发送**：Inject 节点输出 `true` 到 **TRIGGER**，串口按当前 **Value** 发送固定指令。
- **动态内容**：上游字符串节点连到 **VALUE**，每次更新自动发送新内容。
- **外部 OSC 控制**：向 `/dataflow/Show/12/value` 写入 `"01 FF 00"`，再向 `/dataflow/Show/12/send` 发命令，即可远程发十六进制串口包。
