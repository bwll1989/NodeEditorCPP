# Extract 节点

## 1. 节点说明

Extract 从上游 VariableData（键值对象）中**提取字段或计算结果**。支持 JavaScript 表达式：将整份输入映射为 `input` 及顶层键名，执行表达式后输出单个值（数字、字符串、布尔或嵌套对象等）。

表达式为空时，**原样透传**整份输入数据。

## 2. 端口说明

### 输入

| 端口 | 名称 | 数据类型 | 说明 |
|------|------|----------|------|
| 0 | （默认） | VariableData | 待提取的键值数据 |

### 输出

| 端口 | 名称 | 数据类型 | 说明 |
|------|------|----------|------|
| 0 | （默认） | VariableData | 表达式结果；表达式错误或空输入且无数据时可能为空 |

## 3. 界面说明

- **单行文本框**：填写 JS 表达式，编辑完成后（失焦）刷新输出。  
  占位提示示例：`input['key']` 或直接写顶层键名。

表达式环境中：

- `input` — 完整输入对象
- 输入的每个**顶层键**也会注入为全局变量，便于写 `myKey.subField`

## 4. 使用说明

1. 将包含多字段的 VariableData 接到输入。
2. 在文本框写表达式，例如：`input["brightness"]`、`payload.value * 2`。
3. 留空表达式则输出与输入相同的完整对象。
4. 输出接下游单一参数节点。

表达式错误时会在日志中提示，输出为空 VariableData。

配置保存在 `values.key`。

## 5. 示例

**取 OSC 某一参数：** 输入为 `{ "address": "/light", "value": 128 }`，表达式 `input["value"]` 输出 128。  
**透传调试：** 表达式留空，输出接 Data Info 查看完整结构。
