# DelayNode 使用说明

## 用途
DelayNode 用于“延迟触发”：当 TRIGGER 输入收到一次触发时，节点会按界面里配置的延迟列表，在不同时间点触发不同的输出端口。

## 端口
- 输入（1）
  - TRIGGER（端口 0）：VariableData，触发一次延迟队列
- 输出（2，端口可编辑）
  - OUTPUT 0：VariableData（携带最近一次触发时的输入载荷）
  - OUTPUT 1：VariableData（携带最近一次触发时的输入载荷）

说明：
- 输出端口用于“发出 dataUpdated 事件”，从而触发下游节点执行。
- 输出数据内容为最近一次触发时的输入数据（透传载荷）。

## 节点界面
- 延迟列表：为每一条配置“延迟时间（ms）+ 输出端口号”
- ignoreRepeat：是否忽略重复触发

## 使用步骤
1. 在延迟列表中添加多条规则，例如：
   - 100ms → OUTPUT 0
   - 500ms → OUTPUT 1
   - 1000ms → OUTPUT 0
2. 将需要触发的信号连接到 TRIGGER。
3. 将 OUTPUT 0/1 分别连接到需要在对应时刻执行的下游节点。
4. 触发 TRIGGER 后，节点会按延迟列表依次触发对应输出端口。

## 注意事项
- ignoreRepeat 勾选时：若上一轮延迟队列尚未结束，新触发会被忽略。
- ignoreRepeat 不勾选时：新触发会中断当前队列并重新开始计时。
- 未配置任何延迟规则时：触发会立即触发 OUTPUT 0。
