# Delay 节点

## 1. 节点说明

Delay 用于**按时间表延迟触发**下游逻辑。当 TRIGGER 收到一次触发后，节点会按界面中配置的「延迟时间 + 输出端口」列表，在对应时刻依次触发各个输出端口；输出数据为**最近一次触发时**从 TRIGGER 传入的载荷（透传）。

适合场景：分步执行、错峰触发、同一信号在不同时间点驱动不同分支。

## 2. 端口说明

### 输入

| 端口 | 名称 | 数据类型 | 说明 |
|------|------|----------|------|
| 0 | TRIGGER | VariableData | 收到触发后开始执行延迟队列；载荷会被保存并在各输出端口触发时透传 |

输入端口数量默认为 1，**可编辑**（增加输入端口不会改变延迟逻辑，非 TRIGGER 端口仅更新载荷）。

### 输出

| 端口 | 名称 | 数据类型 | 说明 |
|------|------|----------|------|
| 0、1… | OUTPUT 0、OUTPUT 1… | VariableData | 在配置的延迟时刻被触发；数据内容为最近一次 TRIGGER 的输入 |

输出端口数量默认为 2，**可编辑**。端口号在延迟列表的「输出端口」列中指定。

## 3. 界面说明

- **进度条**：显示当前延迟队列的整体进度（0–100%）；空闲时显示「空闲」。
- **延迟列表**：每条规则包含**延迟时间（毫秒）**和**输出端口号**。列表会按时间排序后依次执行；同一时刻的多条规则会同时触发对应端口。
- **忽略重复信号**：勾选后，若上一轮延迟尚未结束，新的 TRIGGER 会被忽略；不勾选则新触发会**中断**当前队列并重新开始。

## 4. 使用说明

1. 在延迟列表中添加规则，例如：100 ms → 端口 0，500 ms → 端口 1。
2. 将需要触发的信号连接到 TRIGGER。
3. 将 OUTPUT 0、OUTPUT 1 等连接到需要在对应时刻执行的下游节点。
4. 触发 TRIGGER 后，节点按时间表依次 `dataUpdated` 对应输出端口。

**行为要点：**

- 未配置任何延迟规则时：触发会**立即**触发 OUTPUT 0。
- 同一时刻可配置多条规则，同时触发多个端口。
- 节点可调整大小；配置会随工程保存（`values` 字段）。

## 5. 示例

**分步灯光控制：** TRIGGER 连接「场景开始」信号；列表配置 0 ms→端口 0（开主灯）、300 ms→端口 1（开辅灯）、800 ms→端口 0（再次触发调光脚本）。勾选「忽略重复信号」可避免演出中途误触打断序列。
