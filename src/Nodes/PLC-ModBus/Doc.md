# PLC-ModBus 节点使用说明

## 用途
通过 Modbus TCP 控制 PLC 的 32 路开关（Z0~Z31），并读取 PLC 的 32 路状态（S0~S31）。

## 端口
节点端口数量可编辑：
- 默认：8 输入 / 8 输出
- 建议需要完整 32 路时：将端口扩展为 32 输入 / 32 输出

### 输入（VariableData）
- Z 0 ~ Z 31：控制输出（bool）

### 输出（VariableData）
- S 0 ~ S 31：状态读取（bool）

## 参数/界面
- Host：PLC/网关 IP（默认 127.0.0.1）
- Port：Modbus TCP 端口（默认 502）
- ServerID：UnitId（默认 1）
- PLCType：Beckhoff / Siemens
- Connect：连接状态显示
- Z0~Z31：控制按钮（用于手动触发）
- S0~S31：状态指示（只读）

## 外部控制（可选）
### 属性（写入）
- /host（string）
- /port（int）
- /serverId（int）
- /plcType（int，0=Beckhoff，1=Siemens）

### 控制（触发/写入）
- /Z0 ~ /Z31（bool）：写入对应 Z 输出

### 反馈（只读）
- /S0 ~ /S31：状态显示（用于外部读取/显示）
- /connect：连接状态

## 地址映射
不同 PLCType 使用不同保持寄存器地址映射（界面标签会同步显示便于现场对照）。

### Siemens
- 控制寄存器（保持寄存器 4x）
  - 连续读取范围：40001..40004（其中 40001 与 40004 有效）
  - Z0~Z15 → 40001.bit0~bit15
  - Z16~Z31 → 40004.bit0~bit15
- 状态寄存器（保持寄存器 4x）
  - 连续读取范围：40007..40012（其中 40007 与 40012 有效）
  - S0~S15 → 40007.bit0~bit15
  - S16~S31 → 40012.bit0~bit15

### Beckhoff
- 控制寄存器
  - 范围：12789..12804（共 16 个寄存器，映射 32 路）
  - 每个地址对应两路：
    - 低位通道：写 bit0（0x0001）
    - 高位通道：写 bit8（0x0100）
- 状态寄存器
  - 范围：13291..13292
  - S0~S15 → 13291.bit0~bit15
  - S16~S31 → 13292.bit0~bit15

## 写入行为（脉冲）
- 当 Zx 被置为 true（来自端口或按钮）：节点会写入对应位为 1。
- 延迟约 500ms：节点会自动回写把该位清 0。

## 注意事项
- 端口数量小于 32 时，只能控制/读取前 N 路（N=当前端口数）。
- Siemens 映射存在“地址空洞”，节点会按实现连续读取中间寄存器但忽略其值。
