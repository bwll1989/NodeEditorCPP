# Merge 节点

## 1. 节点说明

Merge 将**多个输入端口**上的 VariableData，按表格规则**合并为一个**键值对象输出。每一行指定：从哪个输入端口取数、取该端口数据的哪个字段名、以及合并到输出对象时使用的键名（可重命名）。

适合把多路来源拼成一条消息供下游脚本或网络发送。

## 2. 端口说明

### 输入

| 端口 | 名称 | 数据类型 | 说明 |
|------|------|----------|------|
| 0–4（默认 5 个） | INPUT 0 … INPUT 4 | VariableData | 各路人马数据；端口数量**可编辑** |

### 输出

| 端口 | 名称 | 数据类型 | 说明 |
|------|------|----------|------|
| 0 | OUTPUT 0 | VariableData | 合并后的键值对象 |

## 3. 界面说明

表格三列（加操作列）：

| 列 | 含义 |
|----|------|
| ID | 输入端口索引（0、1、2…） |
| Name | 从该端口 VariableData 中要读取的字段名 |
| Rename | 写入输出对象时使用的键名；**留空**则与 Name 相同 |

- **Add Row**：增加合并规则行。  
- 每行 **Delete**：删除该行。

任一输入或表格变更会重新计算输出。

## 4. 使用说明

1. 按需增加输入端口数量。
2. 为每行填写 ID（端口）、Name（源字段）、Rename（可选）。
3. 将多路数据接到对应 INPUT。
4. 从 OUTPUT 0 取合并结果。

若某行 ID 对应端口尚无数据，该行会被跳过。

配置保存在 `values.key`（内含 `rows` 数组）。

## 5. 示例

| ID | Name | Rename | 效果 |
|----|------|--------|------|
| 0 | `x` | `posX` | 端口 0 的 `x` → 输出 `posX` |
| 1 | `y` | `posY` | 端口 1 的 `y` → 输出 `posY` |
| 0 | `label` | （空） | 端口 0 的 `label` → 输出 `label` |

两路位置与一路标签合并后，接 OSC Out 或 JavaScript 节点。
