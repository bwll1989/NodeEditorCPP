# Distribute 节点

## 1. 节点说明

Distribute（与 Inject 同属「Inject」插件）根据**单路输入值**匹配规则表，在命中的输出端口发出布尔 `true` 触发，**不转发**输入的原始数值。适用于一路输入按条件驱动多路分支（如 MIDI 音符、场景号、状态码分发）。

同一输入可匹配**多行**规则，从而同时触发多个输出端口。输出端口数量会随规则中的最大 Port 号自动扩展（默认至少 5 个）。

## 2. 端口说明

### 输入

| 端口 | 名称 | 数据类型 | 说明 |
|------|------|----------|------|
| 0 | INPUT | VariableData | 参与条件匹配的值（通常为主字段 `default`） |

### 输出

| 端口 | 名称 | 数据类型 | 说明 |
|------|------|----------|------|
| 0 … N | OUTPUT 0、OUTPUT 1 … | VariableData | 规则命中时为 `true`；未命中过的端口读取为 `false` |

输出端口**可编辑**；规则表中最大 Port + 1 决定最少端口数。

命中时输出附带：`_pulse`（递增）、`_distributedPort`、`_matchedRow`。

## 3. 界面说明

规则表三列：

| 列 | 说明 |
|----|------|
| Condition | 匹配条件，见下表 |
| Port | 命中后输出 `true` 的端口索引 |
| Action | 删除该行 |

- **Add Rule**：新增规则。  
- 默认示例含 `== 0` 与 `*` 等行，可按需修改。

**条件写法（与 Value Lookup 相同）：**

| 写法 | 含义 |
|------|------|
| `*` | 任意输入均匹配 |
| `45` 或 `== 45` | 等于 45 |
| `> 10`、`< 5`、`>=`、`<=`、`!=` | 数值比较（两侧均为数字时） |

自上而下扫描**所有**行，所有命中行都会触发对应 Port。

## 4. 使用说明

1. 配置 Condition 与 Port 列。
2. 将待分发的值接到 INPUT。
3. 将各 OUTPUT 接到 Inject 的 TRIGGER 或仅需布尔触发的下游。
4. 无规则命中时不更新输出。

规则保存在 `rules`：`[{ "condition": "...", "port": "0" }, ...]`。

## 5. 示例

| Condition | Port | 效果 |
|-----------|------|------|
| `45` | `0` | 输入 45 → OUTPUT 0 为 true |
| `45` | `1` | 输入 45 → OUTPUT 1 也为 true |
| `213` | `2` | 输入 213 → OUTPUT 2 为 true |

一路 MIDI 音符接入 INPUT，多路 OUTPUT 分别启动不同媒体或灯光场景。
