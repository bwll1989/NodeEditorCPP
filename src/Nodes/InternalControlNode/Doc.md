# InternalControlNode 使用说明

## 用途
InternalControlNode 用于维护一组“内部命令（OSC 消息模板）”，并在触发时依次执行这些消息。常用于把一组内部控制动作打包成一个按钮/一个触发点。

说明：
- 该节点触发的是内部解析流程（走内部总线），用于控制应用内部节点与状态。

## 端口
- 输入（1）
  - TRIGGER（端口 0）：VariableData；为 true 时触发一次发送
- 输出：无

## 节点界面
- 消息列表：添加/删除/排序（支持拖拽）
- Trigger 按钮：手动触发一次

## 外部控制（可选）
- `/trigger`：触发一次（bool/脉冲，true 触发）

## 使用步骤
1. 在列表中添加多条消息（每条包含地址与参数）。
2. 用按钮节点/热键节点/条件节点等去触发 TRIGGER。
3. 触发后会按列表顺序逐条执行。

## 注意事项
- 地址请使用以 `/` 开头的路径格式。
- 触发会发送列表中的全部消息；如只想发某一条，建议拆成多个 InternalControlNode。
